# Use Cases

## Описание

Файл содержит пример описания вариантов использования системы BARB — онлайн-системы записи клиентов на услуги сети барбершопов.

В качестве основного примера выбран UC-02 `Забронировать услугу мастера`, так как он отражает центральный пользовательский сценарий системы: клиент самостоятельно выбирает услугу, мастера, свободный слот времени и оформляет запись через сайт.

Для демонстрации связей между вариантами использования дополнительно описаны:

- UC-02.1 `Выбрать услугу мастера и слот` — обязательный подвариант использования;
- UC-02.2 `Указать данные незарегистрированного клиента` — расширяющий подвариант использования.

## Состав вариантов использования

| ID | Название | Основной актор | Назначение |
|---|---|---|---|
| UC-02 | Забронировать услугу мастера | Клиент | Оформить запись на выбранную услугу, мастера и свободный слот времени. |
| UC-02.1 | Выбрать услугу мастера и слот | Клиент | Выбрать тип услуги, конкретную услугу, мастера и свободный слот времени. |
| UC-02.2 | Указать данные незарегистрированного клиента | Незарегистрированный клиент | Указать данные для связи при оформлении записи без регистрации или авторизации. |

## Связи между вариантами использования

| Основной UC | Связь | Связанный UC | Условие / причина |
|---|---|---|---|
| UC-02 | include | UC-02.1 | Выбор услуги, мастера и слота является обязательной частью бронирования. Без этого основной UC не может быть успешно завершён. |
| UC-02 | extend | UC-02.2 | Выполняется только в том случае, если запись оформляет незарегистрированный или неавторизованный посетитель сайта. |

# UC-02. Забронировать услугу мастера

## Краткое описание

Вариант использования описывает взаимодействие клиента с системой онлайн-записи для выбора типа услуги, конкретной услуги, мастера и свободного слота времени.

Запись может выполнить как зарегистрированный, так и незарегистрированный посетитель сайта.

В данном UC не рассматриваются:

- оплата;
- выполнение услуги;
- отзывы клиента;
- фактическая отправка напоминаний.

## Основное действующее лицо

Клиент барбершопа.

Клиент является инициатором взаимодействия с системой, потому что он выбирает услугу и бронирует удобное время записи.

## Цель UC

Обеспечить клиенту возможность самостоятельно выбрать услугу, мастера и свободное время, чтобы записаться на приём через сайт без обращения к менеджеру.

## Интерес действующего лица

Клиент заинтересован в том, чтобы быстро выбрать подходящую услугу, мастера и удобное время, а затем получить подтверждение записи.

## Заинтересованные стороны и их интересы

| Стейкхолдер | Интерес |
|---|---|
| Руководство сети барбершопов | Расширить клиентскую базу за счёт онлайн-записи и снизить количество ручной работы сотрудников. |
| Менеджер | Получать уже оформленные записи клиентов в системе и меньше обрабатывать заявки вручную. |
| Мастер | Видеть актуальные записи клиентов на свои услуги и понимать свою будущую загрузку. |

## Предусловия

1. Система онлайн-записи доступна клиенту.
2. В системе есть актуальное расписание услуг мастеров.
3. В системе есть хотя бы одна услуга, доступная для онлайн-записи.
4. В системе есть свободные слоты для записи.

## Триггер

Клиент решает записаться на услугу через сайт барбершопа.

## Основной сценарий

| Шаг | Действие |
|---|---|
| 1 | Клиент открывает онлайн-запись на сайте барбершопа. |
| 2 | Система запускает UC-02.1 `Выбрать услугу мастера и слот`. |
| 3 | Система показывает клиенту выбранную услугу, мастера и слот времени. |
| 4 | Система определяет данные клиента для оформления записи. |
| 5 | Клиент подтверждает бронирование выбранной услуги, мастера и слота. |
| 6 | Система подтверждает доступность выбранного слота и полноту данных клиента. |
| 7 | Система создаёт запись клиента на выбранную услугу. |
| 8 | Система показывает клиенту подтверждение успешной записи. |

## Альтернативные сценарии

### 4А. Запись оформляет незарегистрированный клиент

1. Система определяет, что клиент не зарегистрирован или не авторизован.
2. Система запускает UC-02.2 `Указать данные незарегистрированного клиента`.
3. Система возвращает выполнение к шагу 5 основного сценария.

### 5А. Клиент отменяет бронирование

1. Клиент отменяет бронирование выбранной услуги.
2. Система не создаёт запись клиента.
3. Система показывает клиенту последнее подтверждённое состояние записи.
4. Вариант использования завершается.

### 6А. Система не подтверждает доступность выбранного слота

1. Система сообщает клиенту, что выбранный слот недоступен.
2. Система возвращает выполнение к UC-02.1 `Выбрать услугу мастера и слот`.

### 6Б. Система не подтверждает полноту данных клиента

1. Система сообщает клиенту, какие данные необходимо уточнить.
2. Клиент передаёт системе уточнённые данные.
3. Система возвращает выполнение к шагу 6 основного сценария.

### 7А. Система не создаёт запись

1. Система сообщает клиенту, что запись не создана.
2. Система сохраняет последнее подтверждённое состояние записей.
3. Клиент повторяет бронирование или завершает работу с UC.
4. При повторном бронировании система возвращает выполнение к шагу 6 основного сценария.

## Постусловия

### Минимальные гарантии

1. Система сохраняет последнее подтверждённое состояние записей.
2. При неуспешном завершении слот не бронируется частично.
3. Клиент получает информацию о результате бронирования.

### Гарантии успеха

1. В системе создана запись клиента на выбранную услугу.
2. За клиентом закреплён выбранный мастер и выбранный слот времени.
3. Система сохраняет контактные данные клиента для связи.
4. Клиент получает подтверждение успешной записи.

## Связанные варианты использования

| Связь | Связанный UC | Описание |
|---|---|---|
| include | UC-02.1 `Выбрать услугу мастера и слот` | Обязательная часть бронирования услуги. |
| extend | UC-02.2 `Указать данные незарегистрированного клиента` | Условный сценарий для записи без регистрации или авторизации. |

# UC-02.1. Выбрать услугу мастера и слот

## Краткое описание

Вариант использования описывает выбор клиентом типа услуги, конкретной услуги, мастера и свободного слота времени для записи.

## Основное действующее лицо

Клиент барбершопа.

## Цель UC

Обеспечить клиенту возможность выбрать подходящую услугу, мастера и свободное время для дальнейшего бронирования.

## Интерес действующего лица

Клиент заинтересован в том, чтобы подобрать удобную услугу, мастера и время записи.

## Заинтересованные стороны и их интересы

| Стейкхолдер | Интерес |
|---|---|
| Менеджер | Чтобы клиент выбирал только доступные услуги и корректное время записи. |
| Мастер | Чтобы клиент записывался только на те услуги, которые мастер действительно выполняет. |
| Руководство сети барбершопов | Чтобы онлайн-запись работала без лишних ручных уточнений. |

## Предусловия

1. В системе есть актуальные услуги.
2. В системе есть актуальное расписание мастеров.
3. В системе есть свободные слоты для записи.

## Триггер

Клиент начинает выбор услуги для записи.

## Основной сценарий

| Шаг | Действие |
|---|---|
| 1 | Клиент выбирает тип услуги. |
| 2 | Система показывает клиенту список услуг выбранного типа. |
| 3 | Клиент выбирает конкретную услугу. |
| 4 | Система показывает клиенту мастеров, которые выполняют выбранную услугу. |
| 5 | Клиент выбирает мастера. |
| 6 | Система показывает клиенту свободные слоты выбранного мастера. |
| 7 | Клиент выбирает свободный слот времени. |
| 8 | Система подтверждает выбор услуги, мастера и слота. |
| 9 | Система передаёт выбранные данные в UC-02 `Забронировать услугу мастера`. |

## Альтернативные сценарии

### 2А. В системе нет услуг выбранного типа

1. Система сообщает клиенту, что услуги выбранного типа недоступны.
2. Клиент выбирает другой тип услуги или завершает работу с UC.
3. При выборе другого типа услуги система возвращает выполнение к шагу 2 основного сценария.

### 4А. В системе нет мастеров для выбранной услуги

1. Система сообщает клиенту, что для выбранной услуги нет доступных мастеров.
2. Клиент выбирает другую услугу или завершает работу с UC.
3. При выборе другой услуги система возвращает выполнение к шагу 3 основного сценария.

### 6А. В системе нет свободных слотов у выбранного мастера

1. Система сообщает клиенту, что у выбранного мастера нет свободных слотов.
2. Клиент выбирает другого мастера или другую услугу.
3. При выборе другого мастера система возвращает выполнение к шагу 5 основного сценария.
4. При выборе другой услуги система возвращает выполнение к шагу 3 основного сценария.

### 8А. Система не подтверждает выбранный слот

1. Система сообщает клиенту, что выбранный слот больше недоступен.
2. Клиент выбирает другой свободный слот.
3. Система возвращает выполнение к шагу 8 основного сценария.

## Постусловия

### Минимальные гарантии

1. Система сохраняет последнее подтверждённое состояние данных.
2. При неуспешном завершении выбор клиента не используется для бронирования.
3. Клиент получает информацию о невозможности выбора, если подходящих вариантов нет.

### Гарантии успеха

1. Клиент выбрал тип услуги.
2. Клиент выбрал конкретную услугу.
3. Клиент выбрал мастера.
4. Клиент выбрал свободный слот времени.

## Связь с другими UC

UC-02.1 включается в UC-02 `Забронировать услугу мастера`.

Тип связи: `include`.

# UC-02.2. Указать данные незарегистрированного клиента

## Краткое описание

Вариант использования описывает ввод контактных данных посетителем сайта, который оформляет запись без регистрации.

Этот UC выполняется только для незарегистрированного клиента.

## Основное действующее лицо

Незарегистрированный клиент.

## Цель UC

Обеспечить возможность оформить запись без создания личного кабинета, сохранив данные, необходимые для связи с клиентом.

## Интерес действующего лица

Незарегистрированный клиент заинтересован в том, чтобы быстро записаться на услугу без обязательной регистрации.

## Заинтересованные стороны и их интересы

| Стейкхолдер | Интерес |
|---|---|
| Менеджер | Иметь контактные данные клиента для ручной связи при необходимости. |
| Мастер | Видеть информацию о клиенте в записи на услугу. |
| Руководство сети барбершопов | Не терять клиентов, которые не хотят проходить регистрацию. |

## Предусловия

1. Клиент не авторизован в системе.
2. Клиент оформляет запись на услугу.
3. Клиент выбрал услугу, мастера и свободный слот.

## Триггер

Система определяет, что запись оформляет незарегистрированный клиент.

## Основной сценарий

| Шаг | Действие |
|---|---|
| 1 | Система запрашивает у клиента данные, необходимые для связи. |
| 2 | Клиент передаёт системе данные, необходимые для связи. |
| 3 | Система подтверждает полноту данных клиента. |
| 4 | Система сохраняет данные клиента в записи. |
| 5 | Система возвращает выполнение в UC-02 `Забронировать услугу мастера`. |

## Альтернативные сценарии

### 2А. Клиент отменяет ввод данных

1. Клиент отменяет ввод данных для связи.
2. Система не сохраняет данные клиента.
3. Система сообщает клиенту, что запись без данных для связи не может быть завершена.
4. Вариант использования завершается.

### 3А. Система не подтверждает полноту данных клиента

1. Система сообщает клиенту, какие данные необходимо уточнить.
2. Клиент передаёт системе уточнённые данные.
3. Система возвращает выполнение к шагу 3 основного сценария.

## Постусловия

### Минимальные гарантии

1. Система сохраняет последнее подтверждённое состояние записи.
2. При неуспешном завершении данные клиента не используются для бронирования.
3. Клиент получает информацию о результате ввода данных.

### Гарантии успеха

1. Клиент указал данные, необходимые для связи.
2. Система сохранила данные клиента в записи.
3. Клиент может продолжить бронирование без регистрации.

## Связь с другими UC

UC-02.2 расширяет UC-02 `Забронировать услугу мастера`.

Тип связи: `extend`.

Условие расширения: клиент оформляет запись без регистрации или авторизации.

## Вывод

В файле показан один основной Use Case и два связанных подварианта.

UC-02 демонстрирует полный сценарий бронирования услуги мастера. UC-02.1 показывает обязательную часть сценария через связь `include`. UC-02.2 показывает условное расширение через связь `extend`.

Такой объём достаточен для портфолио: файл показывает умение описывать акторов, цели, предусловия, триггеры, основной сценарий, альтернативные сценарии, минимальные гарантии, гарантии успеха и связи между вариантами использования.
