# Data Model

## Описание

Файл содержит описание логической модели данных системы BARB — онлайн-системы записи клиентов на услуги сети барбершопов.

Модель описывает основные сущности предметной области, ключевые данные, CRUD-проверку и связи между сущностями.

Подробное описание предметной области находится в `domain.md`, требования к системе — в `requirements.md`.

## Состав модели

В модели данных рассматриваются сущности, которые нужны для поддержки основных функций системы:

- онлайн-запись клиентов;
- работа с зарегистрированными и незарегистрированными посетителями;
- управление услугами;
- управление расписанием мастеров;
- отправка напоминаний;
- фиксация оказанной услуги;
- фиксация оплаты;
- сбор отзывов;
- электронный учёт санитарной обработки;
- управление пользователями, ролями и правами доступа.

## Основные сущности

| Сущность | Описание |
|---|---|
| Клиент | Зарегистрированный посетитель, который использует систему для записи, получения скидок и напоминаний. |
| Посетитель | Незарегистрированный пользователь, который может просматривать услуги и расписание, записаться или пройти саморегистрацию. |
| Мастер | Сотрудник барбершопа, оказывающий услуги и просматривающий своё расписание. |
| Менеджер | Сотрудник, управляющий услугами, расписанием мастеров, записями и шаблонами сообщений. |
| Администратор | Пользователь, управляющий ролями, правами доступа и технической поддержкой системы. |
| Услуга | Услуга барбершопа, на которую клиент может записаться. |
| Тип услуги | Категория услуги, например парикмахерская услуга или косметология. |
| Расписание | График работы мастеров и доступных временных интервалов для записи. |
| Запись | Онлайн-запись клиента или посетителя на услугу, мастера, дату и время. |
| Напоминание | Автоматическое сообщение клиенту о предстоящей записи. |
| Канал связи | Способ отправки уведомлений клиенту: Telegram, WhatsApp, VK или СМС. |
| Оплата | Данные о приёме оплаты за оказанную услугу и передаче данных для учёта. |
| Отзыв | Оценка и комментарий клиента после получения услуги. |
| Санитарная обработка | Данные об обработке помещения по санитарным требованиям. |
| Роль | Набор прав доступа пользователя в системе. |
| Пользователь системы | Учётная запись пользователя, работающего с системой. |

## Словарь ключевых данных

| Данные | Назначение |
|---|---|
| Данные клиента | Используются для записи, истории посещений, скидок и напоминаний. |
| Контактные данные | Используются для связи с клиентом и отправки напоминаний. |
| Канал связи | Определяет способ отправки уведомления. |
| Тип услуги | Используется для группировки услуг. |
| Услуга | Используется при записи, оплате, отчётах и отзывах. |
| Мастер | Используется при выборе исполнителя услуги и формировании расписания. |
| Расписание мастера | Используется для отображения занятости и свободных интервалов. |
| Запись клиента | Фиксирует услугу, мастера, дату, время и статус записи. |
| Напоминание | Используется для уведомления клиента о предстоящей записи. |
| Оплата | Используется для учёта оказанной услуги и передачи данных в бухгалтерию. |
| Отзыв | Используется для оценки качества услуги. |
| Санитарная обработка | Используется для электронного учёта и отчётности. |
| Пользователь, роль и права доступа | Используются для разграничения доступа в системе. |

## Связи между сущностями

| Связь | Описание | Кратность |
|---|---|---|
| Клиент — Запись | Клиент может иметь несколько записей на услуги. | 1:M |
| Посетитель — Запись | Незарегистрированный посетитель может создать запись с минимальными данными. | 1:M |
| Запись — Услуга | Одна запись относится к одной выбранной услуге. | M:1 |
| Тип услуги — Услуга | Один тип услуги объединяет несколько услуг. | 1:M |
| Запись — Мастер | Одна запись закрепляется за одним мастером. | M:1 |
| Мастер — Расписание | У мастера есть расписание и интервалы занятости. | 1:M |
| Расписание — Запись | Запись занимает один доступный временной интервал. | 1:M |
| Запись — Напоминание | По записи может быть создано одно или несколько напоминаний. | 1:M |
| Канал связи — Напоминание | Напоминание отправляется через выбранный канал связи. | 1:M |
| Запись — Оплата | После оказания услуги по записи фиксируется оплата. | 1:1 |
| Запись — Отзыв | После оказания услуги клиент может оставить отзыв. | 1:0..1 |
| Менеджер — Расписание | Менеджер создаёт и обновляет расписание мастеров. | 1:M |
| Менеджер — Услуга | Менеджер создаёт и обновляет услуги. | 1:M |
| Менеджер — Оплата | Менеджер фиксирует оплату за оказанную услугу. | 1:M |
| Администратор — Пользователь системы | Администратор управляет учётными записями пользователей. | 1:M |
| Роль — Пользователь системы | Одна роль может быть назначена нескольким пользователям. | 1:M |
| Санитарная обработка — Менеджер | Менеджер ведёт электронный учёт санитарной обработки. | M:1 |

## Вывод

Модель данных BARB строится вокруг записи клиента на услугу. Ключевыми сущностями являются клиент, посетитель, мастер, услуга, расписание, запись, напоминание, оплата и отзыв.

Дополнительные сущности — пользователь, роль, администратор и санитарная обработка — поддерживают управление доступом, отчётность и регуляторный контур системы.
