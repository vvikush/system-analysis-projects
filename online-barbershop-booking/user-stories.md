# User Stories

## Описание

Файл содержит пользовательские истории для системы онлайн-записи клиентов на услуги барбершопа.

Пользовательские истории описаны для трёх ролей:

- менеджер;
- клиент;
- барбер.

Для историй указаны потребности пользователей, ценность, критерии приёмки, тестовые сценарии, результаты обсуждения и проверка качества по 3C и INVEST.

## Роли

### Менеджер

Менеджер в системе онлайн-записи отвечает за актуальность расписания мастеров, обработку записей клиентов, фиксацию оказанных услуг, оплату и передачу данных в бухгалтерию.

### Клиент

Клиент использует систему для самостоятельной онлайн-записи на услугу, выбора мастера и времени, а также для получения напоминаний о предстоящем визите.

### Барбер

Барбер использует систему для просмотра своего расписания, записей клиентов и передачи данных об оказанной услуге.

## Пользовательские истории

| ID | Роль | Пользовательская история | Ценность |
|---|---|---|---|
| US-01 | Менеджер | Как менеджер, я хочу изменять расписание мастеров и свободные интервалы, чтобы поддерживать актуальные записи клиентов и избегать конфликтов по времени. | Снижение ошибок при записи и поддержание актуального расписания для клиентов. |
| US-02 | Менеджер | Как менеджер, я хочу отмечать выполнение услуги и принимать оплату, чтобы корректно вести учёт оказанных услуг и передавать данные об оплате в бухгалтерию. | Корректный учёт выполненных услуг и оплат. |
| US-03 | Клиент | Как клиент, я хочу выбрать услугу, мастера и удобное свободное время на сайте, чтобы записаться на услугу без звонка администратору. | Возможность записаться на услугу самостоятельно и без звонка. |
| US-04 | Клиент | Как зарегистрированный клиент, я хочу получать напоминание о записи через выбранный канал связи, чтобы не забыть о визите. | Снижение риска неявки клиента. |
| US-05 | Барбер | Как барбер, я хочу просматривать своё расписание и записи клиентов, чтобы планировать рабочий день и заранее подготовиться к услугам. | Возможность видеть актуальные записи и планировать работу. |
| US-06 | Барбер | Как барбер, я хочу передать данные об оказанной услуге, чтобы менеджер мог корректно закрыть визит и зафиксировать оплату. | Корректное закрытие визита и фиксация результата услуги. |

## Критерии приёмки

### US-01 — Управлять расписанием мастеров

| Given | When | Then |
|---|---|---|
| Менеджер авторизован в системе, в системе есть список мастеров и их расписание. | Менеджер изменяет рабочее время мастера или свободный интервал. | Система сохраняет изменения и показывает клиентам только актуальные интервалы для записи. |
| У мастера уже есть запись клиента на определённое время. | Менеджер пытается изменить расписание так, что запись попадает в недоступное время. | Система предупреждает менеджера о конфликте в расписании. |
| Менеджер изменил расписание мастера. | Клиент открывает форму онлайн-записи. | Клиент видит обновлённые свободные интервалы. |

### US-03 — Записаться на услугу онлайн

| Given | When | Then |
|---|---|---|
| Клиент находится на сайте барбершопа, в системе есть услуги, мастера и свободные интервалы. | Клиент выбирает услугу, мастера, свободное время и подтверждает запись. | Система создаёт запись, показывает клиенту подтверждение и делает выбранный интервал недоступным для повторной записи. |

### US-05 — Просматривать своё расписание

| Given | When | Then |
|---|---|---|
| Барбер авторизован в системе, у него есть назначенные записи. | Барбер открывает раздел своего расписания. | Система показывает только записи этого барбера с датой, временем, услугой, именем клиента и статусом записи. |

## Тестовые сценарии

### US-02 — Фиксация выполненной услуги и оплаты клиента

**Название сценария:** фиксация выполненной услуги и оплаты клиента.

#### Предусловия

1. Менеджер авторизован в системе.
2. В системе есть запись клиента на услугу.
3. Клиент пришёл на визит.
4. Мастер оказал услугу.
5. В системе указана стоимость услуги.

#### Тестовые данные

| Поле | Значение |
|---|---|
| Клиент | Иван Петров |
| Услуга | Мужская стрижка |
| Мастер | Алексей Смирнов |
| Дата и время | 03.05.2026, 15:00 |
| Стоимость | 1500 рублей |
| Способ оплаты | Банковская карта |

#### Действия

| № | Действие |
|---|---|
| 1 | Менеджер открывает запись клиента. |
| 2 | Менеджер нажимает «Отметить услугу выполненной». |
| 3 | Система меняет статус записи на «Услуга выполнена». |
| 4 | Менеджер вводит сумму оплаты. |
| 5 | Менеджер выбирает способ оплаты. |
| 6 | Менеджер подтверждает оплату. |
| 7 | Система сохраняет данные об оплате. |

#### Ожидаемый результат

| № | Ожидаемый результат |
|---|---|
| 1 | Статус записи изменён на «Услуга выполнена». |
| 2 | Сумма и способ оплаты сохранены в системе. |
| 3 | Данные об оплате доступны для передачи в бухгалтерию. |
| 4 | Услуга отображается в списке выполненных услуг. |

### US-04 — Получение напоминания о записи

#### Предусловия

- клиент зарегистрирован;
- у клиента есть подтверждённая запись;
- выбран канал уведомления;
- менеджер настроил расписание напоминаний.

#### Действия

1. Система проверяет ближайшие записи.
2. Система отправляет клиенту напоминание через выбранный канал.

#### Ожидаемый результат

- клиент получает напоминание с датой, временем, услугой и мастером;
- в системе фиксируется факт отправки уведомления.

### US-06 — Передача данных об оказанной услуге

#### Предусловия

- барбер авторизован;
- у барбера есть запись клиента;
- услуга фактически оказана.

#### Действия

1. Барбер открывает запись.
2. Барбер отмечает услугу как выполненную.
3. Барбер отправляет данные в систему.

#### Ожидаемый результат

- система сохраняет данные об оказанной услуге;
- запись становится доступной менеджеру для фиксации оплаты и закрытия.

## Обсуждение и согласование

В рамках обсуждения были рассмотрены пользовательские истории для менеджера, клиента и барбера.

| ID | Что обсуждалось | Итог |
|---|---|---|
| US-01 | Нужно ли менеджеру только смотреть расписание или ещё редактировать его. | Решено, что ценность истории именно в редактировании расписания, так как менеджер отвечает за актуальность записей. |
| US-02 | Нужно ли разделять выполнение услуги и оплату на две истории. | Решено оставить вместе, потому что менеджер проставляет выполнение услуги, принимает оплату и передаёт данные в бухгалтерию. |
| US-03 | Достаточно ли указать просто «записаться онлайн». | Уточнено, что клиент должен выбрать услугу, мастера и свободное время. |
| US-04 | Нужно ли отправлять напоминания всем клиентам. | Уточнено, что автоматические напоминания отправляются зарегистрированным клиентам через выбранный канал связи. |
| US-05 | Что именно должен видеть мастер. | Уточнено, что мастер должен видеть своё расписание и записи на свои услуги. |
| US-06 | Для чего барбер передаёт данные об оказанной услуге. | Уточнено, что данные нужны менеджеру для закрытия визита и фиксации оплаты. |

## Проверка 3C

### Card

Пользовательские истории оформлены в компактном формате и описывают отдельные потребности пользователей.

### Conversation

Истории были обсуждены с точки зрения пользователя, заказчика и разработчика. По результатам обсуждения были уточнены действия пользователей, ценность историй и ограничения.

### Confirmation

Для историй указаны проверяемые результаты через критерии приёмки или тестовые сценарии.

## Проверка INVEST

| Критерий | Проверка |
|---|---|
| Independent | Большинство историй может быть реализовано отдельно, но часть историй имеет логические зависимости. |
| Negotiable | Истории обсуждались и уточнялись по результатам согласования. |
| Valuable | Каждая история содержит понятную ценность для пользователя или бизнеса. |
| Estimable | Истории описывают конкретное поведение системы. |
| Small | Истории не перегружены и разделены по отдельным функциям. |
| Testable | Для историй указаны проверяемые результаты, критерии приёмки или тестовые сценарии. |

## Порядок реализации

Рекомендуемый порядок реализации с учётом зависимостей:

1. US-01 — управлять расписанием мастеров.
2. US-03 — записаться на услугу онлайн.
3. US-05 — просматривать своё расписание.
4. US-02 — фиксировать выполнение услуги и оплату.
5. US-04 — получить напоминание о записи.
6. US-06 — передать данные об оказанной услуге.
