# Business & System Analysis Portfolio

Репо содержит учебные pet-проекты по бизнес- и системному анализу, выполненные для демонстрации практических навыков аналитика.

Основной фокус репозитория — анализ предметной области, выявление и структурирование требований, моделирование процессов, описание пользовательских сценариев и подготовка аналитической документации.

## Предметные области

В репозитории представлены материалы по двум проектным задачам:

- система онлайн-записи в барбершоп;
- система доставки заказов.

Подробное описание каждой предметной области и состава материалов находится в README внутри соответствующих папок.

Репозиторий включает аналитические артефакты, которые используются на разных этапах анализа и проектирования IT-систем:

## Ключевые блоки анализа

### Подготовка аналитического пространства

В проектах формировалась рабочая структура аналитических материалов: определялись источники информации, велись глоссарии, фиксировались термины предметной области и подготавливались базовые справочники для дальнейшей работы.

Также применялась декомпозиция для разделения сложных процессов, ролей и действий на более простые и управляемые элементы.

### Работа со стейкхолдерами

В рамках анализа определялись заинтересованные стороны, их роли, интересы, потребности и проблемы.

Для структурирования стейкхолдеров использовались:

- stakeholder registry;
- Onion Diagram;
- таблицы интересов, потребностей и проблем;
- описание ролей и зон ответственности.

### Анализ требований

В проектах прорабатывались бизнес-требования, пользовательские потребности и функциональные требования.

Отдельное внимание уделялось связи между проблемами стейкхолдеров, целями системы и функциями, которые должна поддерживать будущая IT-система.

### As-Is / To-Be моделирование

Для анализа текущего и целевого состояния использовались As-Is и To-Be модели.

В рамках этого блока описывались:

- текущие действия ролей;
- существующие проблемы;
- целевые действия в системе;
- ожидаемые изменения после внедрения решения;
- соответствие функций системы проблемам и потребностям пользователей.

### Контекст системы и потоки данных

Для определения границ системы и её взаимодействия с внешней средой строились контекстные диаграммы.

Также описывались входящие и исходящие потоки данных между системой, пользователями, внешними организациями и смежными IT-системами.

### Доменная и информационная модель

В проектах выделялись доменные сущности, их ключевые атрибуты и связи между ними.

Для проверки полноты действий над сущностями использовались CRUD-матрицы. Для описания структуры данных создавались словари данных, ER-диаграммы и логические модели данных.

### Моделирование процессов

Для описания бизнес-процессов и пользовательских сценариев применялись разные виды диаграмм и таблиц:

- BPMN-диаграммы;
- диаграммы потоков данных;
- swimlane-диаграммы;
- диаграммы состояний;
- таблицы решений.

Также проводилась проверка согласованности артефактов: ролей, требований, процессов, глоссариев, контекстных диаграмм и моделей данных.

### User Stories

Для описания пользовательских потребностей использовались User Stories.

Для них подготавливались:

- краткие описания пользовательской ценности;
- критерии приёмки;
- тестовые сценарии;
- результаты согласования и уточнения пользовательских историй.

### Use Cases

Для формализации взаимодействия пользователей с системой описывались Use Cases.

В рамках Use Case-документации фиксировались:

- акторы;
- цели;
- предусловия;
- триггеры;
- постусловия;
- основной сценарий;
- альтернативные сценарии;
- исключительные сценарии;
- связи include и extend.

## Цель репозитория

Цель репозитория — показать практические навыки бизнес- и системного анализа на примере прикладных проектных задач.

Репозиторий демонстрирует умение:

- анализировать предметную область;
- выявлять и структурировать требования;
- определять роли и стейкхолдеров;
- описывать бизнес-процессы;
- моделировать данные;
- определять границы системы;
- описывать пользовательские сценарии;
- формализовать требования через таблицы, диаграммы и текстовые артефакты;
- подготавливать документацию, понятную бизнесу, разработчикам и другим участникам проекта.
