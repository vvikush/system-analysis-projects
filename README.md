# Business & System Analysis Portfolio

Репо содержит pet-проекты по бизнес- и системному анализу, выполненные для демонстрации практических навыков аналитика.

Основной фокус репозитория — анализ предметной области, выявление и структурирование требований, моделирование процессов, описание пользовательских сценариев и подготовка аналитической документации.

## Предметные области

В репозитории представлены материалы по двум проектным задачам:

- система онлайн-записи в барбершоп;
- система доставки заказов.

Подробное описание каждой предметной области и состава материалов находится в README внутри соответствующих папок.

## Что содержится в репозитории

Репозиторий включает аналитические артефакты, которые используются на разных этапах анализа и проектирования IT-систем:

- описание предметной области;
- анализ стейкхолдеров;
- выявление интересов, потребностей и проблем пользователей;
- формирование бизнес- и функциональных требований;
- As-Is / To-Be моделирование;
- описание границ системы и потоков данных;
- моделирование данных;
- описание бизнес-процессов;
- подготовка пользовательских историй;
- описание вариантов использования;
- визуализация решений через аналитические диаграммы.

### Диаграммы и модели

В проектах используются следующие виды диаграмм и моделей:

- Onion Diagram;
- Context Diagram;
- ER Diagram;
- Data Flow Diagram;
- Swimlane Diagram;
- State Diagram;
- BPMN Diagram.

## Ключевые блоки анализа

### Работа со стейкхолдерами

В проектах определялись ключевые участники процесса, их роли, интересы, потребности и проблемы. Для структурирования использовались stakeholder analysis, Onion Diagram и таблицы потребностей.

### Анализ требований

В рамках проектов выделялись бизнес-требования, функциональные требования, пользовательские потребности и ограничения предметной области. Отдельное внимание уделялось связи между проблемами пользователей и функциями будущей системы.

### As-Is / To-Be моделирование

Для описания текущего и целевого состояния процессов использовались As-Is и To-Be модели. Это позволило показать, какие действия выполняются вручную, какие проблемы возникают в текущем процессе и как система должна изменить работу участников.

### Моделирование данных и процессов

В проектах выделялись ключевые сущности предметной области, их атрибуты и связи. Для визуализации использовались ER Diagram, Context Diagram, Data Flow Diagram, Swimlane Diagram, State Diagram и BPMN Diagram.

### Пользовательские сценарии

Для описания взаимодействия пользователей с системой применялись User Stories и Use Cases. В них фиксировались пользовательские цели, критерии приёмки, акторы, предусловия, триггеры, постусловия, основные и альтернативные сценарии.

## Навыки

Репозиторий показывает навыки:

- анализа предметной области;
- структурирования требований;
- работы со стейкхолдерами;
- описания бизнес-процессов;
- моделирования данных;
- определения границ системы;
- описания пользовательских сценариев;
- подготовки аналитической документации в понятном для команды формате.
