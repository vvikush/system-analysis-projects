# Data Model

## Описание

Файл содержит описание логической модели данных системы DEL — онлайн-системы управления доставкой заказов от поставщиков до конечных клиентов.

Модель данных построена на основе:

- списка сущностей;
- CRUD-матрицы;
- словаря данных;
- ER-диаграммы.

В этом файле описаны только сущности и данные, которые представлены в модели данных. Процессы, требования и стейкхолдеры описаны в отдельных файлах.

## Состав модели

В модели данных DEL представлены следующие сущности:

- Поставщик;
- Точка выдачи;
- Заказ;
- Статус доставки;
- Курьер;
- Бронирование заказа;
- Начисления курьеру.

## Основные сущности

| Сущность | Назначение | Ключевые атрибуты |
|---|---|---|
| Заказ | Описывает задачу по доставке товаров от точки выдачи до конечного клиента. | ID заказа, описание содержимого, адрес доставки, желаемое время доставки, стоимость доставки |
| Точка выдачи | Описывает конкретное физическое место, откуда курьер должен забрать товары для выполнения заказа. | ID точки выдачи, ID поставщика, адрес, часы работы, контактное лицо |
| Статус доставки | Хранит перечень статусов доставки, используемых для контроля выполнения заказа. | ID статуса, наименование |
| Начисления курьеру | Фиксирует сумму, заработанную курьером за выполнение одного заказа. | ID начисления, ID заказа, ID курьера, сумма, дата начисления, статус выплаты |
| Поставщик | Организация, которая является источником заказов и товаров для доставки. | ID поставщика, наименование организации, юридическая информация, контактный телефон |
| Курьер | Исполнитель, который забирает заказ в точке выдачи и доставляет его клиенту. | ID курьера, ФИО, контактный телефон, текущий статус |
| Бронирование заказа | Фиксирует событие, когда конкретный курьер выбирает себе конкретный заказ для последующего выполнения. | ID бронирования, ID заказа, ID курьера, время бронирования |

## CRUD-матрица

| Сущность | Create | Read | Update | Delete / Archive |
|---|---|---|---|---|
| Заказ | Оператор вносит новый заказ. | Курьер, диспетчер и оператор просматривают информацию о заказе. | Диспетчер или курьер корректирует детали заказа и изменяет статус доставки. | Диспетчер отменяет заказ, изменяя статус доставки на «Отменён». |
| Точка выдачи | Администратор добавляет новую точку при регистрации поставщика. | Курьер, диспетчер и оператор просматривают адрес и детали точки. | Администратор изменяет данные точки: адрес и часы работы. | Гипотеза: администратор деактивирует точку без физического удаления. |
| Статус доставки | Гипотеза: администратор заполняет справочник статусов. | Все роли просматривают статусы в заказах. | Гипотеза: администратор изменяет наименование статуса. | Гипотеза: статус не удаляется, чтобы не нарушать историю заказов. |
| Начисления курьеру | Система автоматически формирует начисление после успешной доставки. | Курьер и бухгалтерия просматривают сумму и статус выплаты. | Бухгалтерия изменяет статус выплаты на «Выплачено». | Гипотеза: начисление не удаляется для сохранения финансовой истории. |
| Поставщик | Администратор регистрирует нового партнёра-поставщика. | Диспетчер, оператор и бухгалтерия просматривают информацию о партнёре. | Администратор изменяет реквизиты или контактные данные. | Администратор деактивирует учётную запись при расторжении договора. |
| Курьер | Администратор регистрирует нового курьера. | Диспетчер, администратор и курьер просматривают профиль и статус курьера. | Курьер или администратор изменяет статус, данные профиля. | Администратор деактивирует или блокирует учётную запись при увольнении. |
| Бронирование заказа | Курьер выбирает свободный заказ для выполнения. | Курьер и диспетчер просматривают информацию о бронировании. | Гипотеза: бронирование не изменяется напрямую; для смены курьера бронь отменяется и создаётся новая. | Диспетчер или курьер отменяет бронирование, освобождая заказ. |

## Связи между сущностями

| Связь | Описание |
|---|---|
| Поставщик — Точка выдачи | Поставщик владеет точками выдачи. |
| Точка выдачи — Заказ | Заказ забирается из точки выдачи. |
| Заказ — Статус доставки | Заказ имеет текущий статус доставки. |
| Курьер — Бронирование заказа | Курьер бронирует заказ через сущность бронирования. |
| Заказ — Бронирование заказа | Заказ резервируется через бронирование заказа. |
| Заказ — Начисления курьеру | Заказ порождает начисление курьеру после выполнения доставки. |
| Курьер — Начисления курьеру | Начисление связано с курьером, которому рассчитана сумма. |

## Вывод

Модель данных DEL строится вокруг сущности `Заказ`. Заказ связан с поставщиком через точку выдачи, имеет текущий статус доставки, может быть забронирован курьером и порождает начисление курьеру после выполнения доставки.

Ключевые сущности модели — `Заказ`, `Поставщик`, `Точка выдачи`, `Курьер`, `Бронирование заказа`, `Статус доставки` и `Начисления курьеру`.
