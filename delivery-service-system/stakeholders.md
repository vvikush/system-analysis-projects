# Stakeholders

## Описание

Файл содержит описание стейкхолдеров системы DEL — онлайн-системы управления доставкой заказов от поставщиков до конечных клиентов.

В рамках анализа рассмотрены участники процесса доставки, внешние стороны, регуляторы и участники, влияющие на развитие проекта.

## Проблемы стейкхолдеров

| ID | Стейкхолдер | Действие | Проблемы |
|---|---|---|---|
| ST001 | Команда проекта | Разработка и управление | Риск создать ненужный продукт; ручное управление не работает при росте |
| ST002 | Курьер | Получение заказа; отчёт о доставке | Непрозрачная стоимость заказа; нужно постоянно звонить диспетчеру |
| ST003 | Диспетчер | Распределение заказов; контроль курьеров | Нет информации, где курьер; сложно переназначить проблемный заказ |
| ST004 | Оператор | Ввод заказов | Слишком сложная форма ввода; частые опечатки из-за ручного ввода |
| ST005 | Администратор | Управление доступом | Ручное добавление курьеров занимает много времени; нет уведомлений о сбоях в системе |
| ST006 | Поставщики заказов | Передача заказа | Текущий процесс передачи ненадёжен; непонятно, забрал ли курьер заказ |
| ST007 | Клиенты / получатели | Ожидание заказа | Полная неопределённость по времени; невозможно уточнить детали доставки |
| ST008 | Бухгалтерия | Расчёты | Ручной перенос данных ведёт к ошибкам; сверка данных отнимает много времени |
| ST009 | Спонсоры / инвесторы | Оценка прогресса | Нет прозрачности в успехе проекта; риск потери инвестиций |
| ST010 | Налоговая служба | Регуляторный контроль | Риск некорректных выплат курьерам; неправильная классификация самозанятых |
| ST011 | Конкуренты | Работа на рынке | Появление более дешёвого сервиса; отток курьеров на более выгодные условия |

## Вывод

Ключевыми стейкхолдерами системы DEL являются курьер, диспетчер, оператор, администратор, поставщики заказов, клиенты и бухгалтерия.

Основные проблемы текущего процесса связаны с ручной передачей заказов, постоянными звонками между участниками, отсутствием прозрачности по статусам доставки, сложностью переназначения заказов, ошибками ручного ввода и ручными расчётами.

Целевое состояние предполагает перенос процесса в систему: курьер работает через мобильное приложение, диспетчер контролирует заказы через дашборд, оператор вводит данные через удобный интерфейс, поставщики передают заказы через личный кабинет или API, а бухгалтерия получает автоматизированные отчёты.
